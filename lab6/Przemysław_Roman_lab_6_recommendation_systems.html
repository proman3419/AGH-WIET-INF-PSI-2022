import numpy as np
from surprise import AlgoBase, PredictionImpossible


class ItemAveragePredictor(AlgoBase):
    def __init__(self):
        AlgoBase.__init__(self)

    def fit(self, trainset):
        AlgoBase.fit(self, trainset)

        # mapping: item_id -> average rating
        self.ratings_ = {}
        for k, v in trainset.ir.items():
          self.ratings_[k] = np.mean(np.fromiter(map(lambda x: x[1], v), dtype=float))

        return self

    def estimate(self, u, i):
        if not (self.trainset.knows_user(u) and self.trainset.knows_item(i)):
            raise PredictionImpossible("User and/or item is unknown.")

        return self.ratings_[i]

MAP@k¶

Zdefiniujmy:

  • True Positive (TP) - przedmiot, kt√≥ry naprawdƒô jest w top k (relevant) i kt√≥ry nasz system zarekomendowa≈Ç w top k przedmiot√≥w
  • False Positive (FP) - przedmiot, kt√≥ry nie jest w top k (nonrelevant), ale nasz model go zarekomendowa≈Ç w top k

"Precision at k" to precyzja (precision), obliczona dla top k przedmiot√≥w. Oznaczmy przez $r_k$ liczbƒô TP (relevant items) w≈õr√≥d top k przedmiot√≥w. $$\large P@k = \frac{\text{number of relevant items in top }k}{k} = \frac{r_k}{k} $$

"Average P@k" to po prostu P@k obliczone dla r√≥≈ºnych $i=1,2,...,k$ i u≈õrednione. Taka agregacja bardzo penalizuje umieszczanie nieciekawych przedmiot√≥w na wysokich miejscach, bo uwzglƒôdniamy tylko precyzjƒô dla tych top k, gdzie prawid≈Çowo zauwa≈ºyli≈õmy TP. $$\large AP@k = \frac{1}{r_k} \sum_{i=1}^{k} \left( P@i \text{ if i-th item is relevant} \right) $$

MAP@k to AP@k (average precision at k), u≈õrednione dla wszystkich $|U|$ u≈ºytkownik√≥w: $$\large MAP@k = \frac{1}{|U|} \sum_{u=1}^{|U|}AP@k(u) $$

Im ni≈ºsze $k$, tym surowsi jeste≈õmy i tym ni≈ºsze bƒôdƒÖ wyniki - nasz algorytm musi umie≈õciƒá poprawne predykcje bardzo wysoko, aby uzyskaƒá warto≈õƒá niezerowƒÖ. Typowo $k=5$ lub $k=10$. Zakres warto≈õci MAP@k to $[0, 1]$.

MAP@k przyk≈Çada bardzo du≈ºƒÖ wagƒô do tego, ≈ºeby na pierwszych miejscach trafi≈Çy siƒô jak najlepsze przedmioty. Jest zatem bardzo wa≈ºne, kiedy mamy ma≈Ço miejsc do dyspozycji, np. przy rekomendacji film√≥w na g≈Ç√≥wnej stronie (Netflix).

FCP¶

FCP (Fraction of Concordant Pairs) jest rzadziej u≈ºywanƒÖ, ale bardzo intuicyjnƒÖ metrykƒÖ. IdeƒÖ jest uog√≥lnienie metryki AUROC (ROC AUC) na algorytmy rankujƒÖce, a wiƒôc m.in. systemy rekomendacyjne. Ma zakres warto≈õci $[0, 1]$.

Liczba zgodnych par (concordant pairs) $n_c^u$ dla u≈ºytkownika $u$ to liczba par przedmiot√≥w, kt√≥re zosta≈Çy prawid≈Çowo uporzƒÖdkowane przez ranker. Innymi s≈Çowy, gdy mamy prawdziwy ranking ocen u≈ºytkownika oraz przewidywany, to jest to liczba par przedmiot√≥w, kt√≥re prawid≈Çowo u≈Ço≈ºyli≈õmy (lepszy przedmiot wy≈ºej ni≈º gorszy). $$\large n_c(u) = |\{ (i,j) | \hat{r}_{ui} > \hat{r}_{uj} \text{ and } r_{ui} > r_{uj}\}| $$

Pary niezgodne (discordant pairs) liczy siƒô podobnie: $$\large n_d(u) = |\{ (i,j) | \hat{r}_{ui} > \hat{r}_{uj} \text{ and } r_{ui} \leq r_{uj}\}| $$

Proporcja par zgodnych do wszystkich, zsumowana dla wszystkich u≈ºytkownik√≥w, to FCP: $$\large FCP = \frac{n_c}{n_c + n_d} = \frac{\sum_{i=1}^n n_c(u_i)}{\sum_{i=1}^n \left(n_c(u_i) + n_d(u_i)\right)} $$

Mo≈ºna tak≈ºe obliczyƒá FCP@k, ograniczajƒÖc siƒô do pierwszych k predykcji.

Metryka FCP przyk≈Çada mniejszƒÖ wagƒô ni≈º MAP@k do tego, ≈ºeby najlepsze przedmioty by≈Çy jak najwy≈ºej. Skupia siƒô natomiast na tym, ≈ºeby lepsze przedmioty by≈Çy powy≈ºej gorszych. Dzia≈Ça wiƒôc lepiej dla rekomendacji d≈Çu≈ºszych list, kiedy pierwsze pozycje nie sƒÖ a≈º tak wa≈ºne, np. przy rekomendowaniu playlist muzyki (Spotify).

def ap_k(y_true: List[int], y_pred: List[int], k: int) -> float:
    def r_m(m):
        return len([e for e in y_pred[:m] if e in y_true[:m]])
    r_k = r_m(k)
    res = 0
    if r_k != 0:
        res = sum([r_m(i) / i for i in range(1, k+1) if y_pred[i-1] in y_true]) / r_k
    return res

def fcp_k(predictions: List[Prediction], k: int) -> float:
    top_k = defaultdict(list)
    for uid, iid, true_r, est, _ in predictions:
        top_k[uid].append((iid, est))

    user_item_id_pairs = set()

    for user_id, user_ratings in top_k.items():
        user_ratings.sort(key=itemgetter(1), reverse=True)
        for item_id, rating in user_ratings[:k]:
            user_item_id_pairs.add((user_id, item_id))

    predictions_top_k = [
        pred for pred in predictions if (pred[0], pred[1]) in user_item_id_pairs
    ]

    return fcp(predictions_top_k, verbose=False)

Model średniej bayesowskiej¶

Masz ochotƒô na dobrƒÖ pizzƒô i szukasz opinii na Google Maps. Masz do wyboru 2 lokale: jeden ze ≈õredniƒÖ 5.0 i drugi ze ≈õredniƒÖ 4.8. Zauwa≈ºasz jednak, ≈ºe pierwszy ma tylko 5 opinii, a drugi 200. Kt√≥ry wybierzesz? SƒÖ spore szanse, ≈ºe ten drugi, bo mamy wiƒôkszƒÖ pewno≈õƒá (confidence) co do oceny takiego lokalu.

Prosty model, taki jak ≈õrednia przedmiotu, ma ten sam problem, co powy≈ºej. Sformalizowaniem idei "chcƒô byƒá pewny, ≈ºe ocena przedmiotu jest wysoka" jest model ≈õredniej bayesowskiej (Bayesian average). Mo≈ºliwych sformu≈Çowa≈Ñ bayesowskich jest du≈ºo, ale og√≥lna idea jest zawsze taka, aby wziƒÖƒá pod uwagƒô rozk≈Çad ocen przedmiotu oraz ich liczbƒô. Co wa≈ºne, to dalej sƒÖ rekomendacje globalne - mamy jednƒÖ predykcjƒô per przedmiot.

Czemu ≈õrednia "bayesowska"? Przypomnijmy sobie twierdzenie Bayesa: $$\large P(Y|X) = \frac{P(X|Y) \cdot P(Y)}{P(X)} $$

W naszym wypadku:

  1. $X$ - zbi√≥r danych, kt√≥ry jest sta≈Çy.
  2. $Y$ - przewidywane warto≈õci.
  3. $P(X)$ - prawdopodobie≈Ñstwo zaobserwowania naszych danych, kt√≥re co prawda ciƒô≈ºko jest zmierzyƒá, ale na szczƒô≈õcie w ML zwykle mo≈ºemy zignorowaƒá mianownik, bo to tylko sta≈Ça.
  4. $P(Y)$ - prior (prior distribution), czyli z g√≥ry za≈Ço≈ºony rozk≈Çad prawdopodobie≈Ñstw warto≈õci, kt√≥re przewidujemy. Czƒôsto zaczynamy bez ≈ºadnej wiedzy, wiƒôc zak≈Çadamy rozk≈Çad jednostajny lub normalny.
  5. $P(X|Y)$ - likelihood, wiarygodno≈õƒá, czyli jak dobrze model odwzorowuje dotychczas zaobserwowane dane.
  6. $P(Y|X)$ - posterior (posterior distribution), czyli docelowy rozk≈Çad warto≈õci przewidywanych, obliczony na podstawie danych.

W kontek≈õcie system√≥w rekomendacyjnych:

  • $P(Y)$ (prior) to za≈Ço≈ºony z g√≥ry rozk≈Çad ocen, typowo jednostajny, czyli jest taka sama szansa na ka≈ºdƒÖ ocenƒô
  • $P(X|Y)$ (likelihood) to miara, jak dobrze nasz model odwzorowuje macierz ocen; jakby≈õmy potraktowali go jako skrzynkƒô generujƒÖcƒÖ oceny, to wiarygodno≈õƒá mierzy, jak bliskie sƒÖ te generowane warto≈õci wobec prawdziwych ze zbioru danych
  • $P(Y|X)$ (posterior) to rozk≈Çad przewidywanych ocen dla poszczeg√≥lnych przedmiot√≥w

Jak widaƒá, dostajemy rozk≈Çad w wyniku. Jak dostaƒá konkretnƒÖ predykcjƒô, czyli np. liczbƒô gwiazdek? U≈ºywamy maximum a posteriori (MAP), czyli bierzemy po prostu tƒô ocenƒô, dla kt√≥rej rozk≈Çad posterior ma najwiƒôkszƒÖ warto≈õƒá.

Wykorzystamy podej≈õcie opisane krok po kroku w tym artykule oraz tym tutorialu, w kt√≥rym przewidywana ocena dla $i$-tego przedmiotu (po przekszta≈Çceniach) to: $$\large r_i = \frac{C \cdot m + S_i}{C + c_i} $$

gdzie:

  • $m$ - globalna ≈õrednia ocen dla wszystkich przedmiot√≥w,
  • $C$ - globalna pewno≈õƒá, hiperparametr, przyjƒôta minimalna liczba ocen uwiarygadniajƒÖca ≈õredniƒÖ, mo≈ºe byƒá ustalon np. jako warto≈õƒá pierwszego kwartyla liczby ocen dla wszystkich produkt√≥w w bazie,
  • $S_i$ - suma ocen dla produku $i$,
  • $c_i$ - liczba ocen dla produktu $i$.

Dodatkowe ≈∫r√≥d≈Ça:

  • artyku≈Ç o twierdzeniu Bayesa
  • proste i przyjazne sformu≈Çowanie ≈õredniej bayesowskiej
  • bardziej wyrafinowane podej≈õcie oparte o dolnƒÖ granicƒô b≈Çƒôdu

  • ≈õrednia bayesowska dla danych zmiennych w czasie

class BayesianAveragePredictor(AlgoBase):
    def __init__(self):
        AlgoBase.__init__(self)

    def fit(self, trainset):
        AlgoBase.fit(self, trainset)

        self.global_avg_ = trainset.global_mean

        # mapping: item_id -> sum of ratings
        self.ratings_sums_ = {}
        for k, v in trainset.ir.items():
            self.ratings_sums_[k] = np.sum(np.fromiter(map(lambda x: x[1], v), dtype=float))
        # mapping: item_id -> number of ratings
        self.ratings_counts_ = {k: len(v) for k, v in trainset.ir.items()}
        # confidence (C)
        self.c_ = np.quantile(np.fromiter(self.ratings_counts_.values(), dtype=int), 0.25)

        return self

    def estimate(self, u, i):
        if not (self.trainset.knows_user(u) and self.trainset.knows_item(i)):
            raise PredictionImpossible("User and/or item is unknown.")

        C = self.c_
        m = self.global_avg_
        item_sum = self.ratings_sums_[i]
        item_count = self.ratings_counts_[i]
        
        score = (C * m + item_sum) / (C + item_count)

        return score

Ciƒô≈ºko powiedzieƒá, kt√≥re podej≈õcie jest lepsze, patrz np. ta dyskusja, ten artyku≈Ç. Niezale≈ºnie od podej≈õcia, celem algorytmu jest minimalizacja funkcji kosztu, czyli r√≥≈ºnicy miƒôdzy naszym przybli≈ºeniem $\hat{R} = WU^T$ a prawdziwƒÖ macierzƒÖ $R$. Zwykle wykorzystuje siƒô tutaj b≈ÇƒÖd ≈õredniokwadratowy, w zapisie macierzowym $||\hat{R} - R||_2^2$.

$$\large u_j = \left( \sum_{i \in \Omega_j} w_iw_i^T \right)^{-1} \sum_{i \in \Omega_j} r_{ij}w_i $$

gdzie:

  • $\Omega_j$ oznacza zbi√≥r u≈ºytkownik√≥w, kt√≥rzy ocenili przedmiot $j$.

Mo≈ºna zauwa≈ºyƒá, ≈ºe w obu przypadkach jest to zwyczajny nadokre≈õlony (overdetermined) uk≈Çad r√≥wna≈Ñ postaci $x=A^{-1}b$. Mo≈ºna zauwa≈ºyƒá w powy≈ºszych wzorach, ≈ºe dla $W$ pochodna zale≈ºy od warto≈õci w $U$, a dla $U$ od warto≈õci w $W$ - wydaje siƒô, ≈ºe sytuacja patowa.   2. Powtarzaj przez T krok√≥w:
  3. Zaktualizuj $U$ wed≈Çug wzoru, rozwiƒÖzujƒÖc uk≈Çad r√≥wna≈Ñ; $W$ jest sta≈Çe
  4. Zaktualizuj $W$ wed≈Çug wzoru, rozwiƒÖzujƒÖc uk≈Çad r√≥wna≈Ñ; $U$ jest sta≈Çe

Co wa≈ºne, zbie≈ºno≈õƒá i ALS, i spadku wzd≈Çu≈º gradientu jest gwarantowana, ale do minimum lokalnego. Zwykle nie stanowi to jednak problemu, a w razie czego zawsze mo≈ºna wytrenowaƒá wiele modeli na r√≥≈ºnych random_state i wybraƒá najlepszy.
